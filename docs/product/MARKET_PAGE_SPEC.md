# QuantTrade 行情页专业化规格

日期：2026-07-02

分支：`feature/C-market-page-professional`

## 目标

把当前 C 阶段行情页从基础卡片页升级为中文个人专业看盘工作台。页面借鉴专业行情软件的布局逻辑、图表操作习惯和信息组织方式，不复制任何第三方品牌、素材或受保护界面。

## 范围

本次只补强行情页，不进入 D 阶段 LEAN 回测、策略、组合、交易或 Broker 模块。

P0 范围按四个子任务交付：

1. 行情数据正确性与状态。
2. 主图、周期、复权和指标。
3. 搜索、自选、详情和用户偏好。
4. 视觉收口、性能和 E2E。

## 布局

桌面优先，最低支持宽度 1280px，重点检查 1440x900 和 1920x1080。

页面结构：

```text
顶部导航
行情工作区
├─ 左侧：自选列表，可折叠
├─ 中间：标的标题、工具栏、主图、成交量和副图指标
└─ 右侧：标的详情、数据状态、质量标记，可折叠
```

要求：

- 主图是页面视觉中心。
- 不使用巨型统计卡片堆砌页面。
- 区域边界清晰，间距紧凑但不拥挤。
- 用户切换标的不跳转页面。
- 左右侧栏、周期、复权、图表类型、指标配置等偏好保存到本地版本化结构。

## 数据与状态

行情响应必须明确展示：

- provider；
- market；
- currency；
- resolution；
- adjustment；
- timezone；
- as_of；
- is_delayed；
- stale_age_seconds；
- quality_flags。

页面必须处理：

- 正常；
- degraded；
- unavailable；
- recovering；
- 旧缓存；
- 空数据；
- 请求失败；
- 缺失、重复、乱序、复权因子缺失、交易日历不一致、时区不明确。

旧缓存只允许用于行情展示，必须有全局警告，不得伪装成实时数据。

## 图表

使用 `lightweight-charts` 作为 K 线和成交量主图实现。

P0 图表能力：

- 蜡烛图；
- 折线图；
- 成交量柱；
- 十字光标；
- 鼠标缩放和拖动；
- 一键重置视图；
- 一键全屏；
- 自适应容器；
- hover K 线 OHLCV；
- 切换标的、周期、复权和指标时不重建整页；
- 组件卸载时释放 chart 和监听器。

## 周期

P0 周期：

- 日线；
- 周线；
- 月线。

周线和月线必须由后端正式聚合，前端不得自行把日线随意聚合成正式行情。

## 复权

P0 复权：

- 不复权；
- 前复权；
- 后复权。

图表、标题、详情和接口参数必须语义一致。

## 指标

正式指标由后端计算，前端只选择参数、请求和渲染。

P0 指标：

- SMA；
- EMA；
- MACD；
- RSI；
- 布林带；
- ADX。

默认显示：

- 主图：EMA20、EMA60；
- 副图：成交量；
- MACD、RSI、ADX 默认收起，由用户添加。

指标配置需要支持添加、删除、显示隐藏、参数修改、恢复默认，并保存本地偏好。

## 搜索与自选

搜索 P0：

- 代码和名称搜索；
- 防抖；
- 键盘上下选择；
- Enter 打开；
- 空结果和错误提示；
- 最近访问本地保存；
- 不直接拼 URL。

自选 P0：

- 默认分组“我的自选”；
- 创建、重命名和删除空分组；
- 添加、删除、排序和切换标的；
- 当前标的高亮；
- 本地持久化；
- 刷新恢复；
- 展示最新价和涨跌幅；无可靠数据时显示 `--`。
