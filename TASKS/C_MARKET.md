# 阶段 C：行情与基础看盘

## 阶段目标

完成数据健康治理、A 股搜索、日线、快照、复权、指标、自选和行情页面。

## 去歧义执行规则

- 本文件中的“必须”表示不可跳过。
- 本文件中的“不得”表示禁止实现。
- 未写明的框架、字段、阈值或算法不得由 Codex 自行新增；需要时先记录问题并请求人工决定。
- 简单阶段使用一个分支；复杂阶段仅按本文件列出的分支任务包拆分。

## C-BR1 分支任务包

- **工作分支**：`feature/C-market-backend`
- **人工门禁**：高风险：数据、时区、复权、快照和指标定义必须人工确认
- **状态来源**：`TASKS/INDEX.md`

## 分支级 Loop Engineering

1. 同步最新 `main`，创建本任务包指定分支。
2. 只按本文件中的任务顺序执行。
3. 每完成一个任务：
   - 运行相关测试；
   - 修复全部失败；
   - 更新任务状态；
   - 创建一次清晰 commit。
4. 不为分支内每个任务创建新分支或新 PR。
5. 分支全部任务完成后运行：
   - `make check`；
   - 需要时运行 `make test-integration`；
   - 需要时运行 `make e2e`。
6. 创建一个 PR。CI 失败时继续在原分支修复。
7. 按人工门禁规则合并。
8. 合并后停止，提示用户验收。
9. 只有用户回复“验收通过，继续”后，才将任务包标记 DONE 并领取下一项 READY。
10. 不允许因用户长时间未回复而自动视为验收通过。


### C0：Market 领域、健康状态与质量标记

#### 工作内容

- 定义 Instrument、Bars、Quote、TradingCalendar、DataSourceHealth 和质量标记。
- 实现 healthy/degraded/unavailable/recovering。
- 明确旧缓存只允许用于行情展示；正式快照、回测、风控和下单禁止使用旧缓存。

#### 任务验收

- 健康状态和降级契约测试通过。
- 异常数据不会被当成新鲜数据。
- 相关测试通过后创建一次 commit。

### C1：AKShare、熔断、日线、日历与缓存

#### 工作内容

- 实现搜索、5 年日线、成交量、交易日历和 Redis 性能缓存。
- 连续失败阈值、恢复冷却和最大旧缓存年龄全部从配置读取。
- 降级结果必须返回 is_delayed、as_of、stale_age_seconds 和质量标记。
- 前端可查询数据源健康状态。

#### 任务验收

- 固定标的稳定加载。
- 熔断、恢复和旧缓存标记测试通过。
- 相关测试通过后创建一次 commit。

### C2：数据快照、复权因子与指标

#### 工作内容

- 创建 `data_snapshots` 元数据表。
- OHLCV、公司行动和复权因子保存到 MinIO；Redis 只作为性能缓存。
- 实现 SMA、EMA、MACD、RSI、布林带和 ADX，并固定 Golden 样本。

#### 任务验收

- 快照校验和可验证。
- Redis 丢失后可恢复。
- 指标 Golden 样本通过。
- 相关测试通过后创建一次 commit。

### 分支任务包完成条件

- 本任务包内全部任务完成。
- 完整检查通过。
- PR 描述包含范围、测试、风险、迁移、回滚和未完成项。
- 按门禁规则完成合并。
- 用户验收通过后才能领取下一任务包。
