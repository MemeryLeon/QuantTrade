# 阶段 F：IBKR Paper、风控与对账

## 阶段目标

完成 Paper 账户、订单、Broker 状态新鲜度、风控、恢复、对账和审计。

## 去歧义执行规则

- 本文件中的“必须”表示不可跳过。
- 本文件中的“不得”表示禁止实现。
- 未写明的框架、字段、阈值或算法不得由 Codex 自行新增；需要时先记录问题并请求人工决定。
- 简单阶段使用一个分支；复杂阶段仅按本文件列出的分支任务包拆分。

## F-BR1 分支任务包

- **工作分支**：`feature/F-ibkr-read-model`
- **人工门禁**：高风险：IBKR 字段映射、状态和时效定义需人工确认
- **状态来源**：`TASKS/INDEX.md`

## 分支级 Loop Engineering

1. 同步最新 `main`，创建本任务包指定分支。
2. 只按本文件中的任务顺序执行。
3. 每完成一个任务：
   - 运行相关测试；
   - 修复全部失败；
   - 更新任务状态；
   - 创建一次清晰 commit。
4. 不为分支内每个任务创建新分支或新 PR。
5. 分支全部任务完成后运行：
   - `make check`；
   - 需要时运行 `make test-integration`；
   - 需要时运行 `make e2e`。
6. 创建一个 PR。CI 失败时继续在原分支修复。
7. 按人工门禁规则合并。
8. 合并后停止，提示用户验收。
9. 只有用户回复“验收通过，继续”后，才将任务包标记 DONE 并领取下一项 READY。
10. 不允许因用户长时间未回复而自动视为验收通过。


### F0：Paper 连接、只读同步与新鲜度

#### 工作内容

- 只连接 Paper，隔离凭证并实现健康检查和重连。
- 同步账户、余额、持仓、成交。
- 保存 broker_state_as_of、broker_state_status、connection_status 和最大允许时效。
- IBKR 的断线、超时、限流、会话失效、维护和状态不完整必须转换为类型化 BrokerStateResult。
- 预期网络异常不得直接抛给 RiskService；未预期异常由 Application 边界 Fail Closed。

#### 任务验收

- 断线重连和幂等同步通过。
- 过期状态可被识别。
- 预期 Broker 异常转换测试通过，RiskService 不接触 IBKR SDK 异常。
- 相关测试通过后创建一次 commit。

### F1：订单领域与事件同步

#### 工作内容

- 定义 TradeIntent、BrokerOrder、OrderEvent、Fill 和 idempotency_key。
- 处理乱序、重复和迟到事件。
- 本地状态不得覆盖券商最终状态。

#### 任务验收

- 事件重放一致。
- 相关测试通过后创建一次 commit。

### 分支任务包完成条件

- 本任务包内全部任务完成。
- 完整检查通过。
- PR 描述包含范围、测试、风险、迁移、回滚和未完成项。
- 按门禁规则完成合并。
- 用户验收通过后才能领取下一任务包。

### F2：风控分类、Broker 状态门禁与 Kill Switch

#### 工作内容

- 区分本地确定型规则和 Broker 状态依赖型规则。
- 断线、状态过期、对账未完成或状态未知时，状态依赖型规则必须返回 rejected。
- 稳定错误码使用 `BROKER_STATE_UNKNOWN`。
- 实现跨进程一致 Kill Switch。

#### 任务验收

- 未知状态下不能放行订单。
- Kill Switch 演练通过。
- 相关测试通过后创建一次 commit。

### F3：Paper 下单、撤单、恢复和对账

#### 工作内容

- 实现下单、撤单、重启恢复和券商对账。
- 网络重试不得重复下单。
- 差异不得静默修正。

#### 任务验收

- 重复请求只产生一张订单。
- 差异检测通过。
- 相关测试通过后创建一次 commit。

### F5：断线演练、观察期和阶段验收

#### 工作内容

- 覆盖断线、Worker 重启、API 重试、重复和迟到事件。
- 实现 Paper 观察记录、日终对账和三日汇总报告。
- 系统实际运行至少连续 3 个实际交易日；周末和休市日不计入。
- Codex 不得伪造等待过程或生成虚假观察数据。
- 完成 Phase F 验收。

#### 任务验收

- 未解决订单差异 = 0。
- 未解决成交差异 = 0。
- 未解决持仓差异 = 0。
- 重复订单数 = 0。
- 未解决严重异常数 = 0。
- Kill Switch 演练通过。
- 断线恢复演练通过。
- 生成 `docs/acceptance/F_PAPER_OBSERVATION_REPORT.md`。
- 三个实际交易日数据不足时，任务保持观察中，不得标记完成。
- 相关测试通过后创建一次 commit。
